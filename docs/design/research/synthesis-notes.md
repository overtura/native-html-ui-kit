# Synthesis Notes

## Adopted Principles
- semantic HTML을 component API처럼 다룬다.
- light shell과 neutral border로 반복 예제를 정리한다.
- validation과 focus state는 명확하고 색상에만 의존하지 않는다.

## Rejected Principles
- framework component abstraction은 목적과 맞지 않는다.
- 외부 design system의 exact token/component 이름은 가져오지 않는다.

## Token Changes
- `design-system/base.css`에 native UI kit light token을 둔다.

## Golden Screens
- Menu popover card
- Form validation card
- Dialog progressive enhancement section
