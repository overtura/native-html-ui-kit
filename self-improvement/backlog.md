# Native HTML UI Kit 자가 개선 Backlog

브라우저 기본 HTML 기능을 우선하고 런타임 JavaScript를 추가하지 않는다.

## Planning
- native control 추가 기준과 사용성 기준을 `DESIGN.md`에 정리한다.
- 각 컴포넌트 예제가 보여줘야 할 상태와 fallback 기준을 문서화한다.

## Feature
- 작은 native HTML control 예제를 추가한다.
- 기존 예제의 label, helper text, 상태 설명을 보강한다.

## Design / Visual
- 밝은 UI kit 톤을 유지하며 control spacing과 focus ring을 개선한다.
- 모바일에서 각 예제의 label과 control이 겹치지 않게 다듬는다.

## Accessibility
- form label, fieldset, legend, summary/details 구조를 점검한다.
- keyboard focus와 native validation 메시지 설명을 보강한다.

## Workflow
- no-JS guard와 UI kit 운영 문서의 연결을 개선한다.
